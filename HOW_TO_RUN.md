# دليل تشغيل النظام — HOW TO RUN

> هذا الملف لك أنت فقط. ما يقرأه أي Agent.

---

## 1. هيكل الملفات

```
مشروعك/
│
├── CLAUDE.md                          ← ملف Claude الأصلي (لا تلمسه أبداً ❌)
├── AGENTS.md                          ← تعليمات Codex + المهمة الحالية
├── GEMINI.md                          ← تعليمات Gemini + المهمة الحالية
│
├── .claude/
│   └── rules/
│       └── system-core.md             ← عقل النظام (~60 سطر فقط)
│
├── ai-system/
│   ├── GLOSSARY.md                    ← قاموس المصطلحات
│   ├── PROTOCOLS.md                   ← البروتوكولات التفصيلية
│   ├── TASK_TEMPLATES.md              ← قوالب المهام
│   ├── BRANCH_SUMMARY.md              ← ذاكرة الجلسة
│   ├── ROUTING_LOG.md                 ← سجل قرارات التوجيه
│   ├── FAILURES_AND_REVERTS.md        ← سجل الأعطال
│   └── Learning_Notes.md              ← سجل تعلّمك
│
└── HOW_TO_RUN.md                      ← هذا الملف (لك أنت فقط)
```

---

## 2. قبل أول استخدام

1. انسخ كل الملفات إلى مشروعك بنفس الهيكل
2. **لا تمسح CLAUDE.md** الأصلي الموجود في المشروع
3. افتح `AGENTS.md` وعبّي جدول Project Identity (القسم 1)
4. افتح `GEMINI.md` وعبّي نفس الجدول
5. تأكد إن مجلد `ai-system/` موجود فيه كل الملفات
6. تأكد إن مجلد `.claude/rules/` موجود فيه `system-core.md`

## 4. عبارات تستخدمها مع Claude

| ما تقوله | ماذا يفعل Claude |
|----------|-----------------|
| "الزر ما يشتغل" (أي مشكلة) | يحلل ويكتب Task في ملف Agent |
| "وين وصلنا؟" | يقرأ BRANCH_SUMMARY ويلخصلك |
| "نفّذ أنت هذا" | يتحول لمنفذ بدون Agent |
| "ابعثها لـ Codex" | يكتب Task في AGENTS.md |
| "ابعثها لـ Gemini" | يكتب Task في GEMINI.md |
| "الـ Agent فشل" أو لصق عبارة الفشل | يعيد كتابة Task أوضح |
| "revert" أو "ارجع للخلف" | يساعدك بالـ revert ويسجل |
| "شو تعلمنا؟" | يقرأ Learning Notes ويلخص |
| "شو المشاكل المفتوحة؟" | يقرأ OPEN_ISSUES |
| "حدّث AGENT_PATTERNS" | يراجع ROUTING_LOG ويحدّث |

## 5. عبارات تستخدمها مع Agents

| ما تقوله | ماذا يفعل Agent |
|----------|-----------------|
| (النص اللي أعطاك إياه Claude) | يقرأ الملف وينفذ |
| "وقّف" أو "stop" | يتوقف ويبلغ |

## 6. فهم النتائج

### STATUS — حالة المهمة
| القيمة | المعنى | شو تعمل أنت |
|--------|--------|-------------|
| DONE | تمت بنجاح | انسخ عبارة الإنهاء لـ Claude |
| PARTIAL | تمت جزئياً | انسخ عبارة الإنهاء لـ Claude — هو يقرر |
| FAILED | فشلت | انسخ عبارة الإنهاء لـ Claude — هو يقرر |

### أقسام BRANCH_SUMMARY
| القسم | بالعربي |
|-------|---------|
| CURRENT_STATE | وين المشروع الآن |
| LAST_5_DECISIONS | آخر 5 مهام وش صار فيها |
| OPEN_ISSUES | مشاكل لسه ما انحلت |
| NEXT_TASKS | مهام مقترحة للمرة الجاية |

## 7. حل المشاكل الشائعة

### Agent عدّل ملفات ما طلبت منه
```
قل لـ Claude: "الـ Agent عدّل ملفات زيادة"
أو اعمل: git checkout -- [مسار الملف]
أو اعمل: git revert HEAD
```

### Agent ما التزم بقالب EXECUTION_RESULT
```
عادي — قل لـ Claude: "خلص بس ما التزم بالقالب"
Claude يقدر يقرأ أي شكل
```

### Claude نسي يحدّث الذاكرة
```
قل لـ Claude: "حدّث BRANCH_SUMMARY"
```

### المهمة فشلت مرتين
```
Claude ما رح يرسلها لـ Agent ثاني.
رح يوقف ويسألك شو تبي تعمل.
```

### ما تعرف مين ينفذ
```
قل لـ Claude: "مش عارف مين ينفذ هذا"
هو يقرر بناءً على النظام.
```

## 8. قاموس سريع

| المصطلح | بالعربي |
|---------|---------|
| TASK_ID | رقم المهمة (مثل: 2026-04-04-001) |
| STATUS | حالة المهمة (تمت / جزئي / فشل) |
| TASK ZONE | القسم في الملف اللي فيه المهمة الحالية |
| DONE_IF | متى نعتبر المهمة خلصت |
| DO_NOT_TOUCH | ملفات ممنوع لمسها |
| ESCALATE_IF | متى يتوقف Agent ويبلّغ |
| ISSUES_FOUND | مشاكل اكتشفها Agent بالصدفة |
| BLOCKED_BY | ليش المهمة ما كملت |
| DIFF_LOG | التغييرات في الكود |
| PRE_COMMIT | رقم commit قبل التعديل |
| CONSTRAINT | قيود على الـ Agent |
| COMPLETION_SIGNAL | العبارة اللي يقولها Agent لما يخلص |

## 9. نصائح مهمة

1. **اكتب بس اللي يهم** — Claude يفهم من السياق
2. **لا تخاف تقول "مش فاهم"** — Claude يشرح أبسط
3. **راجع DIFF_LOG دائماً** — أهم شي تعرف شو تغيّر
4. **كل 10 مهام** — قل لـ Claude: "حدّث AGENT_PATTERNS"
5. **لا تعدّل ملفات النظام يدوياً** — خلّي Claude يعدلها
6. **استخدم git بانتظام** — شبكة أمانك
7. **أنت ما تنسخ غير سطرين** — مرة للـ Agent ومرة لـ Claude
